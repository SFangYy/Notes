属性通常用于描述某一类对象拥有的共同特征，为笔记添加属性可以帮助我们==实现有序的分类和高效的检索==。

> 如果你已经很熟悉「属性」的概念的话，可以直接略过本篇。

## Obsidian 中的属性

在 obsidian 中，笔记文件实际上可以分为两个区域，分别是属性区和内容区

我们在笔记中按下 `ctrl+;` （mac 是 `command+;`）快捷键就可以在在属性区里添加一个属性。

每个笔记可以添加多个属性，每一个属性又由 3 部分组成，分别是

- 属性名称
- 属性类型
- 属性值


名称就不多说了，类型的话目前有 6 种，不同的类型决定了我们可以输入的属性值的形式

| 类型    | 说明                                                                      |
| ----- | ----------------------------------------------------------------------- |
| 文本    | 该类型下，属性值可以输入任意的字符，也支持双链语法，比如 \[\[笔记\]\]                                 |
| 列表    | 可以输入多个属性值，每个属性值可以独立编辑，每个属性值的格式等同于文本                                     |
| 数字    | 属性值只能输入数字                                                               |
| 复选框   | 属性值只能是空、 true 或 false，当我们勾选复选框的时候属性值就是 true，取消勾选的时候就是 false，新创建的属性默认值是空 |
| 日期    | 该类型下，属性值的输入形式是一个日期选择器，选择一个日期后就会生成一个日期文本，比如 2024-01-01                   |
| 日期&时间 | 通日期差不多，但是选择的时间单位可以精确到分，生成的属性值格式为： 2024-01-01T12:59                      |

属性描述了这篇笔记的某项特征，而属性值就是对这个特征的详细说明，举个例子： 

我有一篇笔记叫《Obsidian Components 插件》，其中的属性和属性值如下

| 属性名  | 属性值        |
| ---- | ---------- |
| 插件作者 | vran       |
| 最新版本 | 2.0        |
| 插件评分 | ⭐️⭐️⭐️⭐️⭐️ |
| 是否付费 | 是          |

然后有另一篇笔记叫《Obsidian Contribution Graph 插件》，其中的属性和属性值如下

| 属性名  | 属性值        |
| ---- | ---------- |
| 插件作者 | vran       |
| 最新版本 | 1.0        |
| 插件评分 | ⭐️⭐️⭐️⭐️⭐️ |
| 是否付费 | 否          |
也就是说我们不同的笔记可以用相同的属性，但是有不同的属性值。


## 文件属性

除了我们为笔记添加的属性外，系统还会为文件添加一些默认属性，比如

| 属性     | 类型  |
| ------ | --- |
| 文件创建时间 | 时间戳 |
| 文件更新时间 | 时间戳 |
| 文件大小   | 数字  |
这些属性不会在属性区里显示，但是在使用 components 或 dataview 等插件的时候，可以用到这些属性来做筛选和排序。

这里多说一点，不要相信「文件创建时间」这个属性值，因为它是会变的，尤其是涉及到网盘同步等场景时，我们在一台新设备上将笔记下载下来的时候，会发现它的创建时间变成了刚刚下载文件的时间，而不是我们实际第一次创建笔记的时间。

## Components 扩展的属性

Components  扩展了一些内置属性，你可以使用这些属性来对笔记进行筛选统计

> 内置属性就是系统生成的，即使你的笔记中没有显示定义这些属性，也能在 components 的界面看到和使用它们

| 属性名    | 说明                                                             | 示例                              | 备注              |
| ------ | -------------------------------------------------------------- | ------------------------------- | --------------- |
| 文件名    | 文件的名称（不包含文件类型）                                                 | `2024-01-01`                    |                 |
| 文件路径   | 相对于库的文件全路径，包含了文件名称                                             | `journal/2024-W1/2024-01-01.md` |                 |
| 文件父目录  | 文件所在的目录的路径，如果是根目录则返回 `/`                                       | `journal/2024-W1`               |                 |
| 文件大小   | 文件的大小，单位是 Byte                                                 | 1024                            |                 |
| 文件类型   | 文件的扩展名，比如 `xxx.png` 的类型就是 `png`                                | `md`                            |                 |
| 文件标签   | 包含了属性区 `tags` 和内容区内的所有标签，返回的标签列表是以 # 开头的                       | \#journal,\#task                |                 |
| 文件创建时间 | 文件的创建时间，值是时间戳                                                  | 1734424451453                   |                 |
| 文件修改时间 | 文件的最后修改时间，值是时间戳                                                | 1734424451453                   |                 |
| 文件词数   | 不包含属性区的内容词数，一个英文单词算一个词，一个中文字符算一个词                              | 10240                           | 仅 Markdown 文件支持 |
| 文件字符数  | 不包含属性区的内容字符数，但是会包含回车、标点等符号                                     | 10240                           | 仅 Markdown 文件支持 |
| 出链     | 文档中的内部链接列表，每个链接项是原始的链接文本，比如 `[[xxx]]`                          | `[[xxx]]`                       | 仅 Markdown 文件支持 |
| 入链     | 通过链接关联了当前文档的文档列表，每个链接项是一个文件路径， 比如 `area/journal/2024-01-01.md` | `area/journal/2024-01-01.md`    | 仅 Markdown 文件支持 |



有了属性以后我们就可以使用 components 来基于属性对笔记进行管理，比如

- 筛选最近一年读过的书
- 统计今年写的日记的总字数
- 查看去年今日的日记
- ...

这些筛选都是基于笔记的属性来实现的，而且 components 的数据视图比 dataview 更强的一点在于，可以直接在汇总的视图上去编辑属性值。

关于筛选功能会在后续的教程中说明，此处就不再赘述了。
